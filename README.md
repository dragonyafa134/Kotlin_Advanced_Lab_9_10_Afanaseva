# геттеры и сеттеры

Геттер управляет получением значения свойства. Определяется с помощью ключевого слова `get.`
Некоторые особенности работы геттера:
В теле функции описывается, что надо сделать со значением поля, прежде чем его вернуть.
Если достаточно просто вернуть значение как оно есть, явно `get()` можно не писать — геттер создаётся автоматически.
В теле `get()` используется не само имя поля, а ключевое слово field. Это необходимо, чтобы избежать рекурсивных вызовов `get()` — когда из тела функции вызывается эта же функция, из неё опять она же и так далее.
К явному описанию геттера для свойства прибегают, когда значение надо вычислить или как-либо изменить перед его возвратом.

Сеттер определяет логику установки значения переменной. Определяется с помощью слова `set`
Некоторые особенности работы сеттера:
Вызывается, когда свойству присваивается новое значение. Например, `a.number = 34.`
В простейшем случае, который работает по умолчанию, сеттер выглядит так: var number = 0 set(value) { field = value }. Здесь value — параметр функции set(). Вместо value можно использовать любое другое имя переменной. Данному параметру присваивается значение, которое нужно присвоить свойству.
В теле `set()` значение присваивается полю, при этом вместо его настоящего имени (в данном случае number) используется ключевое слово field.

# Data-классы (data class)
Data-класс (класс данных) — это специальный тип классавKotlin,предназначенный в первую очередь для хранения данных. Основное преимуществоdata-классов — автоматическая генерация стандартных методов, которыеобычноприходится писать вручную. Автоматически генерируемые методы:

toString() — возвращает строковое представление объекта;
equals() — сравнивает два объекта на равенство;
hashCode() — возвращает хеш-код объекта;
copy() — копирует данные объекта в другой объект;
componentN() — функции для декомпозиции. Используется для моделей, сущностей, записей, ресурсов, настроек.

# Абстрактные классы и интерфейсы

Абстрактный класс — это базовый класс, от которого нельзя создатьобъектнапрямую. Он используется для описания общей логики и структурыдлягруппысвязанных классов. Может содержать:

* обычные методы с реализацией
* абстрактные методы (без реализации)
* свойства (абстрактные и обычные)
Используется, когда нужно:
* общее поведение и состояние
* базовый класс для иерархии объектов

# Интерфейс (interface)

Интерфейс — это контракт поведения, который класс обязуется реализовать.Особенности:
* не хранит состояние
* может содержать default-реализацию методов
* один класс может реализовывать несколько интерфейсов
